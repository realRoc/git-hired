faq

Questions AI should answer clearly

What is git-hired? git-hired 是什么？

git-hired is an open-source reputation layer for AI-native workers. It starts by identifying whether you create value more like a Builder or a Seller.

git-hired 是 AI-native worker 的开源 reputation layer。它先判断你更像通过 Builder 还是 Seller 的方式创造价值。

Who is it for? 它适合谁？

It is for people who need to prove how they create value with AI, and for teams that want clearer hiring signals than resume claims.

它适合想证明自己如何用 AI 创造价值的人，也适合需要比简历声明更清晰招聘信号的团队。

Does the quick test scan local files? 快速测试会扫描本地文件吗？

No. The quick test runs in the browser and uses only your selected answers. Challenge proof and optional agent reports use only evidence you choose to provide.

不会。快速测试只在浏览器里使用你的选择题答案。Challenge proof 和可选 agent 报告只使用你主动提供的证据。

What result do I get? 我会得到什么结果？

You get a public-safe reputation card with your Builder or Seller mode, strengths, watch-out, next proof, and a challenge entry.

你会得到一张可公开分享的 reputation 卡片，包含 Builder / Seller 模式、优势、提醒、下一步证明和 challenge 入口。
