What The Agent Will Do

agent 会怎么做

  1. Introduce `git-hired` briefly and ask which target role you want right now.
  2. If you already have a clear target role, fetch the matching role prompt and continue directly.
  3. If you do not have a clear target role yet, ask for your current profession or identity first, then route into a universal calibration flow.
  4. Explain the privacy boundary and confirm whether this run stays `history-only` or may inspect only specific repos or files you name explicitly.
  5. Start analysis only after the data boundary is clear.
  1. 先简短介绍 `git-hired`，然后问你现在最想去的目标岗位。
  2. 如果你已经有明确目标岗位，就直接抓取对应 prompt 开始测试。
  3. 如果你还没有明确目标岗位，就先问你当前的职业或身份，再进入通用校准流程。
  4. 说明隐私边界，并确认这次是保持 `history-only`，还是只查看你明确点名允许的 repo 或文件。
  5. 只有在数据边界清楚之后，才开始分析。

内置岗位视角

The current public routes are `AI Agent Engineer`, `Product Manager`, `Global Growth`, and `AI Product Operations`. If your target role does not match these cleanly, the agent falls back to a cross-role calibration first.

当前公开支持的直接路由是 `AI Agent Engineer`、`Product Manager`、`Global Growth` 和 `AI Product Operations`。如果你的目标岗位和这几类对不上，agent 会先走跨岗位校准。

Privacy Boundary

隐私边界

`git-hired` does not upload your local repo or file data to our server. The chosen work agent should inspect only the projects, files, or knowledge-base material you explicitly authorize for this run. If you are using Claude Code or Codex, bypass / YOLO usually makes the run smoother.
`git-hired` 不会把你的本地 repo 或文件数据上传到我们的服务器。你所选的工作 agent 只应访问你这次明确授权的项目、文件或知识库材料。如果你使用的是 Claude Code 或 Codex，bypass / YOLO 通常会让整个流程更顺。
  • Default mode is `history-only`.
  • Specific repo or file access requires explicit in-session approval.
  • Approved scanning should stay on your own machine or connected workspace whenever possible.
  • 默认模式是 `history-only`。
  • 访问指定 repo 或文件必须经过你在会话里的明确授权。
  • 任何批准的扫描都应尽量停留在你的机器本地或已连接工作区内完成。

How To Use It

怎么使用

  • Paste the one-line `read .../skill.md` command into your own work agent.
  • Answer the agent's role and privacy questions honestly.
  • Let the agent fetch the right prompt or route you through universal calibration.
  • Keep the `HIRED` summary and the local markdown report if you decide to apply.
  • 把一行 `read .../skill.md` 命令粘贴到你自己的工作 agent 里。
  • 如实回答 agent 关于岗位和隐私边界的问题。
  • 让 agent 自己抓取对应 prompt，或者带你走通用校准流程。
  • 保留 `HIRED` 结果和本地 markdown 报告；如果之后想申请，可以直接附上。
