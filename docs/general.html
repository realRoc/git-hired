Universal Entry

通用测试入口

Start here if you are switching tracks, sit between roles, or simply are not sure which role page fits best. This page first collects your current profession, target profession, and privacy preference, then generates a prompt for Claude Code, Codex, Notion AI, or any similar work agent with knowledge-base and memory capability.

如果你正在转岗、角色横跨多个方向，或者还不确定该用哪个岗位页，就从这里开始。 这个页面会先收集你当前职业、目标职业和隐私偏好，再生成一段适用于 Claude Code、Codex、Notion AI，或任何具备知识库和记忆能力的工作 agent 的 prompt。

1. Role Context

1. 职业背景

Fill in where you are now and where you want to go. The generated prompt will use this to choose the closest role lens and judge your transition fit.

先填写你现在在做什么，以及你想去做什么。生成后的 prompt 会基于这两点选择最接近的岗位视角，并判断你的转岗信号。

2. Privacy Mode

2. 隐私模式

Choose whether the prompt should stay in `history-only`, or whether you may later authorize access to only specific repos or files. `git-hired` never uploads your local repo or file data to our server.

选择这次测试是保持在 `history-only`，还是你后续愿意按需授权访问指定的本地 repo 或文件。`git-hired` 不会把你的本地 repo 或文件数据上传到我们的服务器。

Friendly tip: if you're using Claude Code or Codex, turning on Claude Code's bypass mode or Codex's YOLO mode usually makes the run smoother.
友情提示：如果你这次使用的是 Claude Code 或 Codex，先开启 Claude Code 的 bypass 模式，或 Codex 的 yolo 模式，通常会让测试过程更顺畅。

What This Universal Entry Does

这个通用入口会做什么

  • Map the candidate's current profession to the closest git-hired lens.
  • Judge whether the target move looks realistic from real work traces, not self-description.
  • Keep the output candidate-facing, local-first, and privacy-bounded.
  • 把候选人当前职业映射到最接近的 git-hired 岗位视角。
  • 基于真实工作痕迹判断目标转向是否靠谱，而不是只听自我描述。
  • 让最终输出保持候选人视角、本地优先、隐私边界明确。
