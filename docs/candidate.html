Candidate Protocol

候选人协议

Run git-hired inside your own work agent, choose your privacy scope, and get a fit report based on how you actually work.

在你自己的工作 agent 里运行 git-hired，选择隐私范围，然后拿到一份基于真实工作方式的匹配报告。

Run It

如何运行

1
Paste the command into Claude Code, Codex, Notion AI, or another work agent with memory.
把命令粘贴到 Claude Code、Codex、Notion AI，或其他具备记忆能力的工作 agent。
2
Choose your target role, or describe your current profession if you are unsure.
选择目标岗位；如果不确定，就先描述你当前的职业或身份。
3
Choose privacy scope: history-only by default, or explicitly named repos / files if you want stronger evidence.
选择隐私范围：默认 history-only，如果你想要更强证据，可以明确点名授权 repo / 文件。
4
Keep the report for reflection, or share a public summary if you want to apply or collaborate.
把报告留作复盘；如果想申请或合作，可以分享公开摘要。
02 /

Privacy Boundary

隐私边界

`git-hired` does not upload local repo or file data to our server. Your work agent should inspect only the projects, files, or knowledge-base material you explicitly authorize.

`git-hired` 不会把你的本地 repo 或文件数据上传到我们的服务器。你的工作 agent 只应访问你明确授权的项目、文件或知识库材料。

03 /

Want To Collaborate?

想合作？

If your report shows strong fit and you are interested in AI-native products, open an issue with your target role, public summary, and what you want to build next.

如果你的报告显示强匹配，并且你对 AI-native 产品感兴趣，可以开一个 issue，写上目标岗位、公开摘要和你接下来想做什么。
