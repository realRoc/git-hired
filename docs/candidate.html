Evidence Levels

证据层级

0
Paste selected text into ChatGPT, Claude, Gemini, or another AI chat. No file access.
把你选择的文本粘贴到 ChatGPT、Claude、Gemini 或其他 AI chat。不访问文件。
1
Use public evidence: GitHub profile, PR, issue, README, project description, or public writing.
使用公开证据：GitHub profile、PR、issue、README、项目说明或公开写作。
2
Approve specific repos, PRs, folders, or files if you want stronger evidence.
如果你想要更强证据，可以明确授权指定 repo、PR、目录或文件。
3
Opt in to local agent history or workspace material only when you are comfortable with that scope.
只有在你明确接受范围时，才授权本地 agent 历史或工作区材料。
02 /

What You Get

你会拿到什么

public
A shareable reputation card: Builder/Seller mode, strengths, challenge proof, anonymized evidence, and risk.
一张可分享 reputation 卡片：Builder / Seller 模式、优势、challenge proof、脱敏证据和风险。
private
A local evidence report with details, confidence limits, weak signals, and next steps for your own review.
一份本地私人证据报告：细节、置信度边界、弱信号和下一步建议。
03 /

Privacy Boundary

隐私边界

No server, no account, no upload, no tracking. Default scope can stay history-only. `git-hired` does not upload local repo or file data to our server. Your work agent should inspect only the projects, files, or knowledge-base material you explicitly authorize.

无服务器、无账号、不上传、不追踪。默认范围可以保持 history-only。`git-hired` 不会把你的本地 repo 或文件数据上传到我们的服务器。你的工作 agent 只应访问你明确授权的项目、文件或知识库材料。

04 /

AI-Native Collaboration

AI-native 协作

If your report or challenge proof shows strong AI-native worker signals and you are interested in AI-native collaboration, open an issue with your target role, public-safe reputation card, and what you want to build or sell next.

如果你的报告或 challenge proof 显示较强的 AI-native worker 信号，并且你正在寻找 AI-native 项目协作，可以开一个 issue，写上目标岗位、可公开 reputation 卡片和你接下来想 build 或 sell 什么。
