# JD Quality Bar

Use this when rewriting or adding role prompts in `git-hired`.

## What good looks like

- concrete role signal, not resume fluff
- behavior-first evaluation, not credential worship
- clear privacy boundary
- startup-fit judgment, not enterprise HR boilerplate
- bilingual parity without line-by-line robotic translation

## Prompt quality rules

- judge from evidence, not vibes
- keep output structures stable
- keep scoring dimensions role-specific
- make risks explicit
- if evidence is thin, lower confidence instead of compensating with prose

## Role-page quality rules

- short top-level explanation
- one-line candidate instruction
- role-specific “what this test looks for”
- embedded prompt must match source prompt files
- no unnecessary navigation clutter

## Anti-patterns

- generic competency lists with no role signal
- asking for private data dumps
- overfitting to keywords instead of behavior
- rewriting every section when only one part changed
- letting English and Chinese versions drift in meaning
